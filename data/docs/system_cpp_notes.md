# 系统级 C++ 代码阅读知识片段

## 指针与所有权
底层 C++ 代码经常使用裸指针表示内存、设备缓冲区或链表节点。阅读时要关注对象由谁创建、谁释放、异常路径是否释放、释放后是否还被访问。

## vector 与数组边界
std::vector 提供 size()，但 operator[] 不进行边界检查。若下标来自地址字段、寄存器编号或用户输入，应显式检查范围。

## bitset 下标
std::bitset<N> 的合法下标是 0 到 N-1。用 bitset 模拟寄存器时，必须说明每一位含义，例如 bit0 表示 zero flag，bit3 表示 exception flag。

## 枚举状态机
系统代码常用 enum 表示状态机，如 IF、ID、EX、MEM、WB，或 INVALID、SHARED、MODIFIED。注释应说明状态迁移条件，而不只是翻译枚举名。

## 日志与断言
assert、日志和异常处理可以帮助定位底层时序问题。实际工程应结合断言、日志和边界测试定位问题。
