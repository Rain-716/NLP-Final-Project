# RAG 混合检索设计说明

本项目采用 Retrieval-Augmented Generation 检索增强生成框架。与普通代码助手不同，系统将底层系统术语、实验手册片段和典型 C++ 代码样例构建为本地知识库。

## 数据处理流程
数据源包括 Markdown 文档、C++ 头文件、实验代码和测试样例。预处理模块先进行编码清洗、换行统一和空白规整，然后根据文件类型选择切分策略。Markdown 文档按标题和段落切分，C++ 文件优先按函数和结构体切分。

## 代码结构切分
普通文本切分可能把一个函数切成上下两半，破坏变量定义和控制流。本项目通过正则识别函数头，再使用花括号匹配寻找函数边界，使一个函数尽量作为完整 chunk 保存。每个 chunk 额外保存 source、line、kind、title、symbols、keywords 等元数据。

## 向量检索
向量检索使用 CodeBERT 将代码和问题映射到语义空间。CodeBERT 对代码标识符、函数结构和自然语言说明都有一定表示能力，因此适合代码问答场景。

## BM25 关键词检索
底层系统代码中，opcode、funct3、dirty、valid、stall 等关键词非常关键。单纯向量检索有时会忽略精确字段名，因此系统加入 BM25 关键词得分。BM25 会提升包含精确术语和变量名的片段。

## 术语扩展
用户可能使用中文“流水线冒险”，代码或文档中可能写成 hazard、RAW、stall、forwarding。系统先识别术语，再把中文词、英文缩写和常见别名拼接到查询中，提高跨语言检索命中率。

## 融合排序
最终得分由三部分组成：CodeBERT 向量相似度、BM25 关键词得分、术语命中得分。默认权重为 0.62、0.30、0.08。这样既保留语义匹配能力，也加强对系统级代码关键词的精确匹配。

## 生成阶段
生成模型接收用户代码、用户问题和 Top-K 检索片段。提示词要求模型输出功能概述、关键变量、执行流程、术语解释、风险与建议，并明确展示检索依据。若本地大模型不可用，系统自动退化到规则生成器，保证演示稳定。
